gRPC API

A gRPC API consists of one or more services and request/response message definitions. A service definition is analogous to a Java interface and is a collection of strongly typed methods. As well as supporting simple request/response RPC, gRPC support streaming RPC. A server can reply with a stream of messages to the client. Alterna- tively, a client can send a stream of messages to the server.

gRPC uses Protocol Buffers as the message format.
